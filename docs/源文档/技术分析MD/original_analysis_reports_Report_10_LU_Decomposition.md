# 第 10 份文件分析报告：LU Decomposition

## 1. 数学核心与 LUP 演进
将方阵 $\mathbf{A}$ 分解为下三角 $\mathbf{L}$ 和上三角 $\mathbf{U}$。实际工程中使用 **LUP 分解**: $\mathbf{PA} = \mathbf{LU}$。
* **部分主元消去:** 交换行确保消元过程中主元尽可能大，极大降低浮点舍入误差。

## 2. 性能与解算
分解复杂度约为 $2/3 n^3$。一旦分解完成，求解 $\mathbf{Ax}=\mathbf{b}$ 转化为 $O(n^2)$ 的前向/后向替换。允许“一次分解，多次求解”。

## 3. 插件中的防御性设计
作为 **Fallback 机制**：优先尝试 Cholesky 获取性能；若美术人员设置导致矩阵非正定，系统自动回退到带主元的 LUP 分解，换取不崩溃的结果。
