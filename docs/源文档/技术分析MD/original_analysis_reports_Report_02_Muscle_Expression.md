# 第 2 份文件分析报告：『鉄拳 8』リアルな筋肉表現をどう実現したか

## 1. 核心挑战与规格演进 (Evolution of Specifications)
在『铁拳 8』中，为了达到次世代的视觉效果，开发团队对骨骼系统的复杂度进行了数量级的提升。这种提升并非简单的堆砌，而是基于**量产化架构**的深度优化。

* **骨骼数量对比 (Bone Count Inflation):**
    * **身体辅助骨 (Body Auxiliary Bones):** 从前作的 **34 根** 增加到 **245 根**（约 7.2 倍）。
    * **面部骨骼 (Face Bones):** 从前作的 **33 根** 增加到 **185 根**（约 6 倍）。
* **架构矛盾：** 极高复杂度的绑定逻辑通常意味着极高的维护成本。文件详细介绍了如何通过“标准化”与“自动化”解决这一瓶颈。

## 2. AnimaDriver：跨平台逻辑解耦架构
为了确保在 DCC 软件（Maya）中设计的肌肉逻辑能够 1:1 还原到游戏引擎（UE5）中，团队开发了 **AnimaDriver** 系统。

* **核心理念：** **“数据驱动的骨骼逻辑”**。
* **解耦机制：** 开发者在 Maya 中使用 AnimaDriver 节点构建肌肉联动逻辑（如：当肱二头肌旋转角度 $\theta > 45^\circ$ 时，触发辅助骨缩放）。这些逻辑被导出为特定的配置文件，UE5 中的运行时组件会读取该文件并重建计算图，从而避免了在引擎中手动重复设置蓝图或代码。

## 3. 关键算法实现：RBF 与 Jiggle Solver
该文件深入探讨了两种支撑次世代肌肉感的核心算法：

* **RBF (径向基函数) 骨骼驱动:**
    用于处理肩膀、大腿等具有复杂自由度（DoF）部位的皮肤形变。通过插值方式，系统能根据四肢的旋转状态平滑地驱动肌肉的隆起与凹陷。
* **Jiggle Solver (动态抖动解算器):**
    实现了基于弹簧-阻尼系统（Spring-Damper System）的实时肌肉震动效果，甚至考虑了慢动作下的 deltaTime 修正。

## 4. 针对量产的“通用权重”策略 (Standardized Weights)
* **方案：** 所有的角色共享同一套辅助骨结构和权重信息。
* **工作流：** 结合 Proportion Rig，辅助骨的位置会随角色体型自动平移，但驱动逻辑和顶点权重保持不变。这使得只需一套肌肉逻辑即可覆盖全员。
