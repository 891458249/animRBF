# 第 3 份文件分析报告：圧倒的キャラクター数×カスタマイズ数をゲームエンジンで実現

## 1. GPU 级别的体型适配架构 (GPU-Level Proportion Transformation)
为了避免对虚幻引擎底层做侵入式修改，同时保证性能，团队通过 C++ 直接操作 GPU 缓存数据。

* **解决方案：** **直接操作渲染数据 (FSkeletalMeshRenderData)**。
    * 架构层在渲染线程中拦截顶点缓冲区（Vertex Buffer）。
    * 根据从 **Proportion Rig** 获取的偏移量参数，实时在 GPU 上对服装顶点的坐标进行重定位（Vertex Pos Overwrite）。
    * **架构解耦：** 逻辑层只需下发一个包含比例缩放系数的 JSON 数据包，底层 C++ 代码自动完成内存映射与显存更新。

## 2. 动态纹理烘焙系统 (Runtime Texture Baking Pipeline)
为了支持高度自由的颜色自定义，系统将各种遮罩、法线贴图和基础色在 GPU 上实时烘焙。

* **优化策略：** 将复杂的材质合成过程“离线化”到游戏加载瞬间。最终材质球只需采样合成后的贴图，极大地降低了战斗时的渲染成本。

## 3. 核心算法：基于几何面积变化的自动皱纹计算 (Automatic Wrinkle Calculation)
使用 Compute Shader 根据面部骨骼蒙皮前后的三角形面积变化（利用**海伦公式**计算）来动态混合法线贴图。

* **数学原理：** 利用海伦公式计算面积 $S = \sqrt{s(s-a)(s-b)(s-c)}$。
* **计算逻辑：**
    1. **原始态：** 记录模型在静止姿态下的每个三角形面积 $S_{base}$。
    2. **运行态：** Compute Shader 计算当前帧后的面积 $S_{runtime}$。
    3. **判定系数 $K = S_{runtime} / S_{base}$：** 若 $K > 1$ 说明拉伸，皱纹减少；若 $K < 1$ 说明挤压，根据程度增加皱纹法线权重。
