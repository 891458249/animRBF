# 第 4 份文件分析报告：鉄拳 8 で進化した補助骨とその応用例の紹介

## 1. 辅助骨系统的“内核”：AnimaDriver 节点算子集
开发团队将辅助骨的驱动逻辑抽象为一套类似虚幻引擎蓝图或 Maya Hypergraph 的节点系统。从架构师视角看，这本质上是一个**有向无环图 (DAG) 演算引擎**。

文件定义了六类核心算子 (Nodes):
1. **Source:** 获取动画骨骼变换。
2. **Target:** 设置辅助骨变换。
3. **Calc:** 执行数学运算（加减乘除、点积叉积）。
4. **Constraint:** 朝向约束 (Aim)。
5. **Solver:** 复杂解算 (RBF, Jiggle)。
6. **Other:** 曲线驱动和权重混合。

## 2. 典型应用案例分析
* **眼睑自动跟随:** 根据眼球转动方向，眼睑通过辅助骨微调形状，实现自然的视线追踪。公式：$\Delta d = f(\vec{v} \cdot \vec{axis}) \cdot Scale_{char}$。
* **部位分离 (Limb Disconnection):** 在肢体断开时复制整套辅助骨逻辑，并将驱动源从“主躯体空间”切换为“脱离肢体局部空间”，确保形变依然自然。

## 3. 高性能优化方案
* **预计算 (Pre-computation):** 静态约束在资产加载时编译为高效线性指令。
* **SIMD 加速:** 底层代码针对 SSE/AVX 指令集优化，确保每帧计算数百根辅助骨时 CPU 开销控制在微秒级。
