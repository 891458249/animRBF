# 第 1 份文件分析报告：『鉄拳 8』 カスタマイズキャラクターのための多重リグシステム

## 1. 核心架构设计：双重绑定系统 (Dual Rig Architecture)
为了解决“高精度肌肉表现”与“高度自由度角色自定义”之间的冲突，系统架构被拆分为两个核心层级：

* **体格调整绑定 (Proportion Rig / 体格調整用リグ):**
    * **职责：** 负责角色基础体型（身高、胖瘦、肢体比例）的变更。
    * **架构特征：** 它是一套处于“隔离层”的骨骼，不直接参与最终渲染，而是作为参数驱动源。它通过 JSON 配置文件定义各角色的体型差异。
* **肌肉表现绑定 (Muscle Rig / 筋肉表現用リグ):**
    * **职责：** 负责战斗中的动画表现、辅助骨骼逻辑以及肌肉的挤压/伸展模拟。
    * **架构特征：** 这是最终交付给渲染管线的骨骼系统，包含了复杂的辅助骨（Auxiliary Bones）逻辑。

## 2. 关键数学模型与技术：P2M (Proportion to Muscle) 机制
系统实现了一个名为 **P2M** 的转换层，其底层逻辑遵循以下数学映射关系：

设 $P_{base}$ 为标准体型骨骼空间，$P_{char}$ 为特定角色体型空间。系统通过线性变换矩阵 $M_{prop}$ 计算偏移：
$$V_{muscle} = M_{prop} \cdot V_{standard} + \Delta Offset$$

* **自动化适配 (Automated Fitting):** 开发者只需针对“标准体型”制作服装或道具。
* **运行时变形 (Runtime Deformation):** 在游戏加载阶段，通过读取 JSON 定义的参数，系统利用 Compute Shader 或 CPU 预计算，将标准资产的顶点信息强行覆盖（Vertex Info Overwrite）到目标体型上，确保无穿模。

## 3. 系统解耦与工程化实践 (Engineering Implementation)
* **解耦逻辑：** 将“骨骼位置偏移（Translation）”与“骨骼旋转（Rotation）”彻底分离。体格调整仅修改 $T$ 通道，而肌肉模拟主要操作 $R$ 和 $S$（缩放）通道。
* **资产流水线：** 这种架构允许角色模型（Modeling）和绑定（Rigging）并行。即使模型比例发生微调，只需更新 Proportion Rig 的参数，无需重新制作全套动画。

---

### 核心技术要点总结 (Technical Highlights):
1. **多重リグ (Multi-Rig):** 解决了《铁拳 7》中角色体型过于统一的问题，支持从庞大的 Jack-8 到娇小的角色的平滑过渡。
2. **量産化 (Mass Production):** 通过 P2M 流程，极大地降低了数百件自定义道具适配不同角色的工作量。
3. **运行时计算 (Runtime Processing):** 利用虚幻引擎（UE）的底层 API，在内存中动态生成各角色特有的骨骼偏移量。
